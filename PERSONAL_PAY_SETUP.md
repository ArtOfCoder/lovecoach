# 个人收款码支付配置指南

## 概述

由于微信小程序的微信支付需要企业资质和商户号，个人开发者无法直接接入。本方案采用**个人收款码 + 确认码**的方式实现付费解锁功能。

## 使用流程（最终版）

1. 用户在小程序中点击"解锁"
2. 弹窗提示用户扫码支付 9.9 元
3. 用户截图保存收款码，用微信扫一扫识别
4. 用户向开发者转账 9.9 元
5. 用户返回小程序，点击"我已支付，上传截图"
6. 用户从相册选择支付截图上传
7. **系统自动解锁**，显示 AI 头像

### 支付遇到问题？

用户可点击页面底部"联系客服"按钮，复制微信号 `news-tomato`，添加好友备注"恋爱小程序"获取帮助。

## 配置步骤

### 1. 准备收款码图片

你需要生成自己的微信收款码：

1. 打开微信 → 我 → 服务 → 收付款
2. 选择"二维码收款"
3. 设置金额：9.9 元（可选）
4. 保存收款码到手机
5. 将收款码图片重命名为 `pay-qr-code.jpg`
6. 放入小程序项目的 `images/` 目录下

### 3. 用户支付后的处理

当用户支付后，你需要：

1. 查看微信转账记录，确认收到 9.9 元
2. 核对订单号备注
3. **无需任何操作**，用户已自动解锁

### 4. 管理后台

访问路径：`pages/admin/admin`

**进入方式：**
1. 在开发者工具中直接访问 `/pages/admin/admin`
2. 或在"我的"页面添加隐藏入口（连续点击版本号等）

**功能：**
- 查看所有支付记录（订单号、金额、时间）
- 查看用户上传的支付截图
- 统计总收入和订单数
- 复制订单号便于核对
- 清空记录

**管理员密码：**`love2026`

### 5. 核对收入

虽然用户上传截图后会自动解锁，但你应该定期：
1. 打开管理后台查看支付记录
2. 点击"查看截图"核对支付凭证
3. 对比微信转账记录确认到账

## 防骗机制

### 1. 截图水印标识
每次支付时生成唯一水印标识（如 `2347A8`），提示用户截图中需包含此标识，便于你核对。

### 2. 截图重复检测
系统自动检测同一张截图是否被重复使用：
- 基于文件大小和修改时间生成唯一特征码
- 同一张截图无法重复解锁

### 3. 支付截图存档
用户上传的支付截图会保存在本地，你可以在管理后台查看：
- 点击"📷 查看截图"查看支付凭证
- 对比微信转账记录确认到账

### 4. 人工核对建议
虽然系统会自动解锁，但建议你定期：
1. 打开管理后台查看支付记录
2. 点击"查看截图"核对支付凭证
3. 对比微信转账记录确认到账
4. 发现异常及时处理

### 5. 如何处理假截图

如果发现用户上传了假截图：

1. **进入管理后台**：`pages/admin/admin`
2. **找到该订单**：在支付记录列表中找到可疑订单
3. **点击"🚫 封禁"按钮**：
   - 输入封禁原因（如：假截图、重复支付等）
   - 确认封禁
4. **封禁效果**：
   - 该用户（基于出生日期）将无法再次解锁
   - 下次尝试支付时会提示"账户受限"
   - 被封禁的用户会看到联系客服的选项

5. **查看黑名单**：
   - 点击"黑名单"标签查看所有被封禁用户
   - 可以查看封禁原因和时间
   - 支持解封操作

## 优缺点分析

### 优点
- ✅ 无需企业资质
- ✅ 无需微信支付商户号
- ✅ 无需服务器和云开发
- ✅ 零成本接入
- ✅ 资金直接进入个人微信钱包
- ✅ 有截图存档，可追溯
- ✅ 重复截图自动拦截

### 缺点
- ❌ 基于信任机制，仍存在伪造截图风险
- ❌ 无法自动验证支付金额和时间
- ❌ 无法处理退款
- ❌ 不适合高并发场景
- ❌ 需要人工定期核对

## 替代方案

如果未来需要更完善的支付体验，可以考虑：

1. **微信云开发支付**
   - 需要：企业资质 + 微信支付商户号
   - 优点：自动处理，用户体验好

2. **第三方聚合支付**
   - 如：PayJS、Ypay 等
   - 需要：个人身份证即可
   - 缺点：有手续费，需要接入 API

3. **赞赏码/小商店**
   - 微信小程序小商店支持个人开店
   - 可以上架虚拟商品
   - 需要一定审核流程

## 注意事项

1. 收款码图片不要上传到公开仓库（GitHub 等）
2. 建议定期更换收款码
3. 保留转账记录，以备查验
4. 如遇恶意退款，需自行承担风险
