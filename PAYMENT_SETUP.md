# 微信支付配置指南

## 概述

小程序已集成完整的微信支付功能，使用微信云开发的 `cloudPay` 能力。按照以下步骤配置后即可上线使用。

## 前置条件

1. 小程序已开通**云开发**（微信开发者工具 → 云开发）
2. 小程序已绑定**微信支付商户号**
3. 商户号已开通**JSAPI支付**权限

## 配置步骤

### 第一步：开通云开发微信支付

1. 登录 [微信云开发控制台](https://console.cloud.tencent.com/tcb)
2. 选择你的小程序环境
3. 进入「云开发」→「设置」→「支付方式」
4. 点击「绑定微信支付商户号」
5. 按照指引完成商户号绑定

### 第二步：配置云函数

#### 1. 部署云函数

在微信开发者工具中：

```
cloudfunctions/create-order/index.js
cloudfunctions/pay-notify/index.js  
cloudfunctions/query-order/index.js
```

右键点击每个云函数文件夹 → 选择「创建并部署：云端安装依赖」

#### 2. 创建数据库集合

在云开发控制台 → 数据库中创建以下集合：

- `orders` - 存储订单信息
- `payment_logs` - 存储支付日志
- `user_payment_stats` - 存储用户支付统计

### 第三步：配置商户信息（如使用服务商模式）

如果使用普通商户模式，云开发会自动处理，无需额外配置。

如果使用服务商模式，修改 `cloudfunctions/create-order/index.js`：

```javascript
const PAY_CONFIG = {
  mchid: '你的商户号',  // 如：1234567890
  appid: '小程序AppID', // 如：wx1234567890abcdef
}
```

## 支付流程

```
用户点击支付
    ↓
调用 create-order 云函数
    ↓
云函数调用微信支付统一下单API
    ↓
返回支付参数给小程序
    ↓
小程序调起微信支付界面 (wx.requestPayment)
    ↓
用户完成支付
    ↓
微信服务器通知 pay-notify 云函数
    ↓
更新订单状态为已支付
    ↓
用户解锁内容
```

## 测试支付

### 开发版测试

1. 在微信开发者工具中开启「真机调试」
2. 使用微信扫码在真机上预览
3. 真机上可以调起支付（需要登录小程序账号）

### 体验版测试

1. 上传代码为体验版
2. 添加体验成员
3. 体验成员扫码进入体验版
4. 测试支付功能

### 沙箱环境（测试用）

如需在开发阶段测试而不产生真实扣款，可使用微信支付沙箱环境：

修改 `cloudfunctions/create-order/index.js`：

```javascript
// 在调用 unifiedOrder 前添加沙箱标志
const res = await cloud.cloudPay.unifiedOrder({
  // ... 其他参数
  sandbox: true, // 启用沙箱模式
})
```

**注意**：沙箱模式仅供开发测试，上线前必须关闭。

## 常见问题

### Q: 支付时提示 "商户号未绑定"
A: 需要在云开发控制台绑定微信支付商户号

### Q: 支付成功但订单状态未更新
A: 检查 `pay-notify` 云函数是否正确部署，以及数据库权限设置

### Q: 开发版无法调起支付
A: 开发版需要使用真机调试，模拟器不支持支付

### Q: 如何退款？
A: 需要在微信支付商户平台操作退款，或添加退款云函数

## 安全注意事项

1. **不要在客户端暴露商户密钥**
2. **务必验证支付回调的签名**
3. **订单金额在服务端计算，不要信任客户端传来的金额**
4. **敏感操作（如发货、解锁）必须在支付回调中处理**
