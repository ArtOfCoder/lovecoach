   - 这些是 Canvas 本地绘图方案遗留，现已改用通义万相 API 生图，不再需要

### 顺带清理
- 删除 `soulmate.wxml` 第 3 行残留的隐藏 Canvas 标签（之前 Canvas 方案遗留）

### 优化结果

- 清理后：~0.49MB（506 KB）
- 远低于微信 2MB 限制

## 修复"保存图片"失败问题

### 问题
生成图片后点"保存图片"，提示"下载失败，请检查网络"

### 根本原因
`downloadAndSaveImage` 已将通义万相图片下载并保存到本地（`wx.env.USER_DATA_PATH/soulmate_xxx.png`），`imageUrl` 是本地 `wxfile://` 路径。但 `saveImage()` 却对这个本地路径再次调用 `wx.downloadFile`，无法对本地路径 downloadFile，导致失败。

### 修复内容
1. **`saveImage()`** — 根据 imageUrl 是否为本地路径（`wxfile://` 开头）来分流：
   - 本地路径：直接 `wx.saveImageToPhotosAlbum({ filePath })`
   - 网络 URL：先 downloadFile 再保存
2. **`downloadAndSaveImage()`** — 先 `unlinkSync` 删除旧文件，避免"文件已存在"报错

## 修复 unlinkSync 产生错误日志问题

### 问题
日志显示 `error occurs: no such file or directory, unlink 'wxfile://usr/soulmate_xxx.png'`，虽然功能正常但日志很碍眼。

### 原因
`unlinkSync` 对不存在的文件抛异常时，微信底层日志系统仍会打印 error，即使 catch 住了也会留痕。

### 修复
改为先 `accessSync` 检查文件是否存在，存在才 `unlinkSync` 删除，不存在则 `accessSync` 抛出，直接忽略，不触发 unlink 报错。
