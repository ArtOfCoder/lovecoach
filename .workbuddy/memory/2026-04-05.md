## 支付防作弊系统（2026-04-05）

### 问题背景
用户担心：
1. 如何确保上传的截图是真正的付款截图
2. 如何防止付款金额与要求不匹配
3. 如何防止用户只付一次却反复使用

### 解决方案

#### 1. 截图哈希检测
- 每张上传的截图生成唯一哈希值（基于文件大小+时间戳）
- 检查该哈希是否已被使用过
- 同一张截图无法重复使用

#### 2. 设备指纹识别
- 收集设备信息（型号、系统、屏幕尺寸等）
- 生成唯一设备指纹
- 用于识别同一设备的多次支付

#### 3. 支付频率限制
- 同一设备最小支付间隔：5分钟
- 同一用户最大支付次数：3次
- 超过限制需要人工审核

#### 4. 风险评分系统
| 风险因素 | 分值 | 处理 |
|---------|------|------|
| 未上传截图 | +30 | 强制要求上传 |
| 短时间多次尝试 | +40 | 额外确认弹窗 |
| 截图重复使用 | +100 | 直接拒绝 |
| 设备环境变化 | +20 | 计入评分 |

- 0-49分：低风险，正常解锁
- 50-99分：中风险，需要额外确认
- 100+分：高风险，直接拒绝并记录

#### 5. 欺诈检测与封禁
- 自动记录欺诈尝试
- 同一用户多次欺诈自动加入黑名单
- 黑名单用户无法使用支付功能

#### 6. 人工审核队列
- 可疑支付进入审核队列
- 保存截图路径和风险评分
- 管理员可在后台查看审核

### 用户界面更新
1. **截图要求提示**
   - 必须包含：支付金额 ¥9.9、支付时间、交易单号
   - 警告：截图将被系统验证，重复使用将被限制

2. **支付流程**

## OCR自动验证系统（2026-04-05）

### 功能概述
用户上传支付截图后，系统自动进行OCR识别验证，无需人工审核。

### 验证流程
1. **频率检查** - 检查支付频率限制
2. **图片哈希** - 生成图片哈希用于去重检测
3. **重复检测** - 检查截图是否已被使用过
4. **OCR识别** - 调用百度OCR API识别截图文字
5. **内容分析** - 分析金额、时间、交易信息
6. **自动解锁** - 验证通过后自动解锁

### 验证规则
| 检查项 | 分值 | 说明 |
|--------|------|------|
| 金额 ¥9.9 | 40分 | 必须正确 |
| 支付时间 | 30分 | 支持多种格式 |
| 交易标识 | 20分 | 交易单号/商户信息等 |
| 支付截图特征 | 10分 | 微信/支付宝标识 |

- **>= 70分**：自动通过，解锁成功
- **< 70分**：验证失败，提示重新上传

### 防作弊机制
1. **截图重复使用检测** - 同一截图无法多次使用
2. **作弊行为记录** - 记录重复上传等作弊行为
3. **自动封禁** - 同一用户作弊3次自动封禁
4. **黑名单系统** - 封禁用户无法使用支付功能

### 新增函数
- `confirmPaid()` - 显示截图上传弹窗
- `submitScreenshotForReview()` - 提交截图进行自动审核
- `recognizeScreenshot()` - 百度OCR识别
- `analyzePaymentScreenshot()` - 分析支付截图内容
- `saveOcrRecord()` - 保存OCR识别记录
- `recordCheatAttempt()` - 记录作弊尝试
- `banUser()` - 封禁用户

### 界面更新
- 新增截图上传弹窗（模态框形式）
- 上传区域支持相册选择和拍照
- 截图预览和重新上传功能
- 截图要求提示列表
- 防作弊警告提示

### 修改文件

需要在 `soulmate.js` 中配置百度OCR API密钥：
```javascript

const BAIDU_SECRET_KEY = '你的百度Secret Key'
```
获取方式：
1. 访问 https://cloud.baidu.com/
2. 注册并登录百度智能云
3. 创建应用，选择"通用文字识别"
4. 获取 API Key 和 Secret Key
